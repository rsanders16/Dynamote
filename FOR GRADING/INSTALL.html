Installing Dynamote

Note: if you are a windows user, then you can simply use Rails Installer and skip to step 4

Step 1: Installing Ruby

Dynamote runs on Ruby on Rails. Rails is a framework that runs on Ruby. You can install Ruby a number of ways, but here is an overview of the steps:

Check your Ruby version

Ruby 1.8.6 comes standard with Mac OSX 10.5 and above. This can be achieved by typing ruby --version in the terminal. If you are running on a windows machine, then you can use ruby installer.

Install Ruby

If ruby is not installed, (ruby --version came up empty) then you can use Ruby Version Manager (RVM) to install. Simply run the following commands (this requires git): $ bash < <(curl -s https://raw.github.com/wayneeseguin/rvm/master/binscripts/rvm-installer) and rvm install 1.9.3. You can check your ruby version again to verify you have ruby 1.9.3.

Step 2: Installing RubyGems

RubyGem is a system that is akin to packages in Java. It is a way to manage libraries that can be used by ruby. To install RubyGems, run the following code in the Terminal:
curl -O http://rubyforge.iasi.roedu.net/files/rubygems/rubygems-1.3.1.tgz
tar xzvf rubygems-1.3.1.tgz cd rubygems-1.3.1
sudo /usr/local/bin/ruby setup.rb
cd ..

Step 3: Installing Rails

Now we are ready to install rails. Rails is just a RubyGem, so it can be installed like this:
sudo gem install rails
This will also install all the libraries it needs to make sure rails works. It also installs a UNIX program called rails. You can type $ which rails to verify.
Also, there is a gem we will need called mysql2. This is gem that helps rails talk to mySQL, which will be set up in step 4.

Step 4: Installing MySQL

Verifying

Dynamote uses MySQL as its primary database. To check if you have MySQL installed already, you can type which mysql or mysql --version it does NOT come pre-installed on mac,windows or most linux distributions.

Downloading

Head to the [MySQL download page](http://dev.mysql.com/downloads/mysql) to download the latest version.

Make sure you use the 32-bit or 64-bit version depending on your hardware architecture.

Installing

Run `mysql-[version number]-[OS]-[hardware architecture].pkg` first, to install MySQL.  You may want to install

The preference pane too, because that makes it easier to activate or deactivate the MySQL server. You may also want to verify MySQL is installed properly. Open up a Terminal, and type $ which mysql to see where the mysql binaries are located. Also verify that this directory is in your PATH by typing $ echo path. This is usually done during the installation.

Setting up MySQL for Dynamote

Type $ mysql -u root in a Terminal window to open the MySQL monitor as a root user (if you set up a root password already, add -p and then type your password.) To see the databases, type SHOW DATABASES; to view what databases are there. If everything looks correct, you are ready to create the database that Dynamote will use.
While still in the MySQL monitor, type CREATE DATABSE dynamote_development;. This creates a container for Dynamote's data. Now we need to create a user for the dynamote app:
GRANT ALL PRIVILEGES ON dynamote_development.* TO 'dynamote_user'@'localhost'
IDENTIFIED BY 'rordm319';
where 'rordm319' is the password used to access dynamote's database as dynamote_user.
Dynamote is almost set up. Open a terminal window to the folder where Dynamote was installed. From here, run the command $ rake db:migrate to run Dynamote's migrations on the mysql server. Migrations are a Rails convention used to set up a database from Ruby files, so that it is set up the same way every time.

Step 5: Running Dynamote

Now, from a Terminal window in the Dynamote directory, type rails server to start a WEBrick server. (WEBrick comes with Rails automatically.) If there were no problems with the setup, you should be able to go to the link provided by WEBrick (usually localhost:8080) to use Dynamote.
